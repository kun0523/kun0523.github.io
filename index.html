Hexo Tutorial

发表于 2018-11-04 | 分类于 tutorial

项目文件解释：

  • Folders：
  1. node_modules:
    存放Node.js的东西，不需要管；
  2. scaffolds:
    存放内容模板，通过scaffold下的文件，可以设置文章中共有的内容；
  3. source:
    存放实际文本内容，这里放 博客.md 草稿.md 页面.md 等文件；
  4. themes:
    存放网页模板，
  • Files:
  1. .gitignore:
    设置git同步时忽略的文件
  2. _config.yml:
    配置文件.

创建新对象：

创建一篇Blog

hexo新建的默认对象是post，这个可以在配置文件里(Writing部分)进行修改。

创建一篇草稿

在正式发布前的草稿内容，可以先新建一个草稿来存放；
hexo new draft test_draft_1

草稿文件在页面上不会显示，如果想要查看草稿的渲染结果：
hexo server --draft

当草稿写好了，要发布草稿，使用：

会发生的行为是，将draft文件夹下的文件移动到post文件夹下。

创建一个网页

创建一个网页：
hexo new page test_page_1

给文章分类和加标签：

  1. 在文章的tag标签中添加不同的tag，可以通过点击tag，访问隶属于不同tag的文章列表；
  2. 给文章添加categories类别标签，类似于tag标签，只是写的方式不同：

便于标识文章中的一些特殊元素，例如代码：
{ % codeblock lang:python % } { % endcodeblock % }

文章中的素材

  1. 在配置文件”_config.yml”中，修改”post_asset_folder”属性：
    post_asset_folder: true
  2. 之后再新建文章时会自动产生一个与文章同名的文件夹，用于存放素材；
  3. 不知道为什么，我的并不成功……

用别人做好的主题：

  1. 在官网上找中意的主题；
  2. 克隆到”themes”文件夹下；
    git clone https://github.com/klugjo/hexo-theme-clean-blog.git themes/clean_blog
  3. 修改配置文件”_config.yml”中，属性”theme”的值，改为刚刚下载的主题名；
    theme: clean_blog
  4. 重启Hexo服务之后即可；

自定义主题：

  1. 新建文件夹：
  • kun_theme
    • languages
    • layout
    • scripts
    • source
    • config.yml
  1. 更改配置文件”_config.yml”中，属性”theme”为自己定义的主题名；
  2. 重启Hexo服务；
  3. 配置各个文件夹下的文件……

部署在GitHub上

  1. 在GitHub上新建仓库，注意仓库名： GitHub用户.github.io !!坑!!;
  2. 安装git的插件：npm install hexo-deployer-git --save;
  3. 修改配置文件，在deploy区域修改属性：type: git, repo: https://github.com/kun0523/kun0523.github.io.git, branch: master;
  4. 根据markdown文件生成静态html文件：hexo generate 也可简写为： hexo g;
  5. 将本地文件部署到github上：hexo deploy 也可简写为：hexo d;
